Sigil Composition Protocol

The user sigil is a deterministic composite SVG generated from three data sources: the genesis seed (cosmologicalId), the user's tier, and their earned achievements. Every layer is algorithmically derived — no choices are made, only revealed.

Layer Architecture

Layer 1 · Core
Familiar Sigil
The inner 56px circle contains the familiar's unique geometric SVG — lines, nodes, and accent paths generated from the SHD-CCP archetype pattern.
Layer 2 · r=36
Tier Ring
Solid ring for INSTRUCTOR, doubled ring for ARCHON, dashed for ACOLYTE. Color encodes rank: gold / cyan / slate.
Layer 3 · r=43
Achievement Ring
Dot markers appear when Dept 7, 8, or 9 are unlocked. White, amethyst, and emerald pips at evenly-spaced angular positions.
Layer 4 · r=48
Guild Ring
One pip per guild affiliation (up to 4), color-coded by guild archetype, placed at equal angular spacing on the ring.
Layer 5 · r=50
Outer Border
Tick marks encode tome and minor-tome count (up to 16 ticks) in purple, forming a progress halo around the full sigil.

PRNG Seeding

The familiar archetype is selected using the xmur3 + sfc32 algorithm — a cryptographic-quality 128-bit seeded PRNG from the SHD-CCP specification. The cosmologicalId (a hex string derived from the genesis position in the SPIRE hyperbolic manifold) is the sole input. The same ID will always produce the same familiar.
